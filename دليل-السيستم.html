تجهيز ManyChat (مرة واحدة بس)

حاجات تعملها مرة واحدة في البداية وتنساها

🔑 أ) هات الـ API Key و الـ Page ID

في ManyChat افتح Settings (الترس فوق)
روح تاب API
انسخ API Key — وحطه في السيستم: الإعدادات ← مفاتيح API ← ManyChat API Key
انسخ Page ID كمان وحطه في نفس المكان
🎯 ليه؟ السيستم محتاج المفتاح ده عشان يقدر يبعت رسائل آلية للعميل (زي رسالة الترحيب بعد الزيارة).

🏷️ ب) اعمل Custom Fields

الـ Custom Fields دي خانات بتتخزن فيها معلومة عن كل عميل. اعملها كده:

Settings → Custom Fields → + New Custom Field
اعمل الخانات دي واحدة واحدة:
اسم الخانةالنوعبتتخزن فيها إيه
campaign_sourceTextاسم الحملة الإعلانية
ad_idTextرقم الإعلان
crm_lead_classTextتصنيف العميل (السيستم بيحدّثه تلقائي)
crm_total_scoreNumberنقاط العميل (السيستم بيحدّثه تلقائي)
💡 فايدة crm_lead_class و crm_total_score السيستم بيرجّعهم لـ ManyChat، فتقدر تعمل Segment للعملاء الـ "hot" بس وتبعتلهم عرض خاص.

ربط الإعلان — نعرف العميل جه منين

عشان جدول "قمع الإعلانات" في التحليلات يشتغل
🎯 المشكلة اللي بنحلها عندك حملة فيها كذا إعلان. عايز تعرف كل إعلان لوحده بيجيب عملاء جادين ولا لأ — عشان توقف الإعلان الفاشل وتزوّد ميزانية الإعلان الناجح. الحل: نسجّل اسم الحملة و رقم الإعلان مع كل عميل من أول لحظة.
⚠️ مهم — إعلاناتك Message Campaign إعلانات الـ Message (اللي بتفتح ماسنجر) مفيهاش روابط Ref URL. الـ Ref URL ده للروابط العضوية بس (بايو/بوست). إعلاناتك بتشتغل بطريقة تانية أسهل ↓

🎯 أ) Trigger لكل إعلان (مش روابط)

لكل إعلان في الحملة، اعمل trigger منفصل في ManyChat:

في الـ Flow اضغط + New Trigger
اختار "User clicks a Facebook Ad"
ManyChat هيوريك قايمة إعلاناتك — اختار الإعلان المحدّد اللي عايز تتبعه
كرّر ده: trigger لكل إعلان في الحملة (إعلان واحد = trigger واحد)
💡 ليه trigger لكل إعلان؟ عشان كده تقدر تحط لكل إعلان اسم الحملة ورقمه بإيدك — وبالتالي في جدول "قمع الإعلانات" هتشوف كل إعلان في الحملة لوحده وتقارنهم.

⚙️ ب) لكل trigger، سجّل الحملة + رقم الإعلان

جوه كل trigger، أول حاجة قبل أي رسالة، حط الـ Actions دي:

Set Custom Field → campaign_source → اكتب اسم الحملة (مثلاً eid_2025) — نفس الاسم لكل إعلانات الحملة دي
Set Custom Field → ad_id → اكتب رقم/اسم الإعلان (مثلاً video_a أو رقم الإعلان من Meta Ads Manager) — مختلف لكل إعلان
بعدها External Request بالـ JSON ده:
POST /api/events — أول دخول من الإعلان
{
  "user_id":         "{{messenger user id}}",
  "first_name":      "{{first name}}",
  "event_type":      "entry_offer",
  "campaign_source": "{{campaign_source}}",
  "ad_id":           "{{ad_id}}",
  "source":          "manychat"
}
📌 مثال — حملة فيها 3 إعلانات
• إعلان 1 → campaign_source=eid_2025 ، ad_id=video_a
• إعلان 2 → campaign_source=eid_2025 ، ad_id=photo_b
• إعلان 3 → campaign_source=eid_2025 ، ad_id=carousel_c
في جدول "قمع الإعلانات" هيظهروا 3 سطور تحت نفس الحملة — كل سطر بيقولك الإعلان ده وصّل كام عميل لطلب الموقع. الإعلان اللي رقمه ضعيف → اوقفه. القوي → زوّد ميزانيته.
⚠️ أهم نقطة في الدليل كله user_id لازم يكون {{messenger user id}} — مش {{user id}}. ده الرقم اللي السيستم بيعرف بيه العميل. لو غلطت فيه، العميل هيتعدّ مرتين ونقاطه متتجمعش صح.

الـ External Request — قلب السيستم

ده اللي بيوصّل خبر من ManyChat للسيستم. هتعمله كتير فاحفظه كويس

الـ External Request هو "تليفون" بين ManyChat والسيستم. كل ما العميل يعمل حاجة مهمة، ManyChat بيتصل بالسيستم ويقوله "العميل ده عمل كذا". بتضيفه كده:

في أي Flow، بعد الزر أو الرسالة، اضغط + Add Action
اختار External Request
Request Type: اختار POST
Request URL: الصق ده:
https://medo-backend-production.up.railway.app/api/events
تاب Headers: اضغط + Request Header وحط Key = Content-Type ، Value = application/json
Header تاني للأمان: Key = x-webhook-secret ، Value = الكود اللي في السيستم (الإعدادات ← مفاتيح API ← فوق في بانر ManyChat فيه "Webhook Secret" — انسخه)
تاب Body: اكتب الـ JSON (هتلاقي كل JSON في الخطوة الجاية)
للـ user_id: امسح القيمة، اضغط {+} Add a Field، اختار Messenger User ID
حط اسمك في خانة Contact for testing واضغط Test Request
شوف تاب Response — لو لقيت "success":true يبقى تمام، اضغط Save
✅ علامة إنه شغّال في الـ Response هيظهرلك حاجة زي: {"success":true,"new_score":20,"lead_class":"cold"}
🔒 الـ Webhook Secret افتح الإعدادات ← مفاتيح API — فوق هتلاقي بانر ManyChat، جواه Webhook Secret. انسخه وحطه في كل External Request كـ header اسمه x-webhook-secret. بعد ما تحطه في كل الـ flows، فعّل webhook_enforce من الإعدادات عشان السيستم يرفض أي طلب مزوّر.
متفعّلهوش قبل ما تحط الـ header في كل الـ flows وإلا هتقف عن الاستقبال.
🟢 تأكيد إن ManyChat LIVE في نفس البانر (الإعدادات ← مفاتيح API) لازم يكون مكتوب ManyChat: LIVE ✅. لو MOCK ⚠️ يبقى الـ API Key مش متحط — مفيش رسائل آلية هتتبعت للعملاء.
📌 مثال يوضّح الفكرة العميل ضغط "عرض تفاصيل انتريه اليار" → ManyChat يتصل بالسيستم ويقوله "العميل رقم ٥٣٧٨٠٩ شاف منتج اسمه اليار من فئة انتريهات" → السيستم يديله ٢٠ نقطة ويسجّلها.

① دخل من الإعلان — entry_offer

اتشرح في الخطوة 3 فوق (مع بيانات الحملة).

② اختار فئة — category_request

لما يضغط "كتالوج غرف النوم" مثلاً. غيّر category حسب الفئة:

POST /api/events — category_request
{
  "user_id":    "{{messenger user id}}",
  "first_name": "{{first name}}",
  "event_type": "category_request",
  "category":   "غرف النوم",
  "source":     "manychat"
}

③ شاف منتج — product_details

لما يضغط "عرض التفاصيل" على منتج. حط اسم الموديل وفئته:

POST /api/events — product_details
{
  "user_id":         "{{messenger user id}}",
  "first_name":      "{{first name}}",
  "event_type":      "product_details",
  "product":         "اليار",
  "category":        "الانتريهات",
  "campaign_source": "{{campaign_source}}",
  "ad_id":           "{{ad_id}}"
}
⚠️ مهم كل منتج له Flow تفاصيل لوحده، فغيّر "product" في كل واحد (اليار / روفان / ...). و"category" لازم تكون بالظبط زي الأسماء في الخطوة 6.

④ طلب الموقع — location_request

لما يضغط "عايز عنوان الفرع" أو "فين الفروع". دي أقوى إشارة شراء:

POST /api/events — location_request
{
  "user_id":         "{{messenger user id}}",
  "first_name":      "{{first name}}",
  "event_type":      "location_request",
  "campaign_source": "{{campaign_source}}",
  "ad_id":           "{{ad_id}}"
}
💡 ليه دي مهمة جداً الـ location_request بيدي +40 نقطة ويخلّي العميل hot على طول. وده العمود اللي بيظهر في جدول "قمع الإعلانات" — كل إعلان وصّل كام عميل للخطوة دي.

⑤ اختار فرع — branch_selected

لما العميل يختار فرع: قبل ما تبعتله العنوان، اطلب منه رقم تليفونه. أول ما يكتبه ابعت الحدث ده ومعاه الرقم — ده اللي الاستقبال هيأكّد بيه الزيارة:

POST /api/events — branch_selected + phone
{
  "user_id":     "{{messenger user id}}",
  "first_name":  "{{first name}}",
  "event_type":  "branch_selected",
  "event_value": "nasr_city",
  "phone":       "{{phone}}"
}

قيم الفروع: nasr_city ، maadi ، new_cairo ، october ، alexandria

📱 الـ phone {{phone}} هو الرقم اللي العميل كتبه. في ManyChat ضيف خطوة User Input → Phone قبل ما تبعت العنوان، واحفظه في حقل، وحطه في "phone". أي صيغة (01… / ‎+20…) السيستم بيظبّطها لوحده.

⑥ أكّد الزيارة — visit_confirmed

اتشرح بالتفصيل في الخطوة 8 (تأكيد الزيارة).

الكاتيجوريز الـ 6 — الأسماء بالظبط

لازم تكتبها حرفياً كده عشان التحليل يتجمّع صح
⚠️ قاعدة حديدية لو كتبت "انتريهات" بدل "الانتريهات" — السيستم هيعتبرهم فئتين مختلفتين والتحليل هيتلخبط. انسخ الاسم من الجدول ده بالظبط.
الفئةالقيمة في الـ JSON (انسخها بالظبط)
🛏️ غرف النوم"category": "غرف النوم"
🪑 غرف السفرة"category": "غرف السفرة"
🛋️ الانتريهات"category": "الانتريهات"
🧸 غرف الأطفال"category": "غرف الأطفال"
🛋️ الركنات"category": "الركنات"
🪞 المنتجات الجانبية"category": "المنتجات الجانبية"

"المنتجات الجانبية" بتجمّع: الترابيزات + الجزامات + وحدات الشاشة.

💡 الـ 6 دول بيظهروا دايماً في صفحة التحليلات حتى لو فاضيين — كل واحد فيه: كام عميل شاف منتجات، كام طلب الكتالوج، كام موديل، وأكتر الموديلات مشاهدةً.

تأكيد الزيارة — برقم التليفون

إزاي تربط العميل الأونلاين بزيارته الفعلية للمعرض
🎯 الفكرة العميل كلّمك أونلاين وقالك هييجي. عشان تعرف إنه جه فعلاً: قبل ما تبعتله عنوان الفرع، اطلب رقم تليفونه. الرقم بيتسجّل في السيستم مع حسابه، ولما ييجي المعرض الاستقبال يكتب نفس الرقم ويتأكّد إنه زار.

① اطلب الرقم قبل العنوان في ManyChat

في الـ Flow اللي العميل اختار فيه الفرع، قبل رسالة العنوان ضيف خطوة User Input → Phone ("ابعتلنا رقم تليفونك عشان نجهّزلك زيارتك 📱")
احفظ الرقم في حقل (مثلاً الحقل الجاهز {{phone}} في ManyChat)
ضيف External Request (حدث branch_selected) ومعاه "phone": "{{phone}}" — زي المثال في الخطوة 5
بعد كده ابعتله رسالة العنوان عادي
💡 مش لازم صيغة معيّنة للرقم. العميل يكتب 01012345678 أو +20… — السيستم بيوحّدهم تلقائياً عشان الاستقبال يلاقيه.

② الاستقبال يأكّد بالرقم

العميل يوصل المعرض، موظف الاستقبال يسأله عن رقم تليفونه
يفتح في السيستم تاب الاستقبال
يكتب رقم العميل ويضغط تأكيد
السيستم يلاقي العميل بالرقم ويحوّله لـ visited تلقائياً
💡 بونص أول ما الاستقبال يأكّد، السيستم بيبعت تلقائياً رسالة ترحيب للعميل على ماسنجر (الـ Visit Flow اللي مظبوط في الإعدادات).

الداشبورد — تقرأ التحليلات إزاي

إيه اللي بيظهرلك بعد ما الداتا تيجي

📊 صفحة التحليلات فيها:

الكروت العلوية: إجمالي العملاء، الساخنين، الزيارات، المشتريات
تحليل الفئات: الـ 6 كاتيجوريز — كل واحدة كام عميل وكام موديل. اضغط أي فئة تشوف موديلاتها
قمع الإعلانات: كل إعلان وصّل كام عميل لحد طلب الموقع (اللي إنت طلبته)
أكثر المنتجات مشاهدةً: الموديلات الأكثر طلباً في كل الفئات
📌 إزاي تقرأ "قمع الإعلانات"
سطر زي ده:
حملة "عيد 2025" — 100 عميل → 60 شافوا منتجات → 25 طلبوا الموقع → 8 زاروا → 3 اشتروا — 25%
معناه: الإعلان ده كفاءته 25% (ربع اللي جابهم وصلوا لمرحلة إنهم عايزين العنوان). قارن الإعلانات ببعض وحط فلوسك في اللي نسبته أعلى.

👥 الداشبورد بيختلف حسب المستخدم

المدير (admin): يشوف كل حاجة — التحليلات، كل العملاء، الإعدادات، دليل ManyChat
المندوب (rep): يشوف بس عملاءه المعيّنين — تابات: مكالمات اليوم / مهامي / عملائي / إنجازاتي / الاستقبال
💡 اعمل مستخدمي المندوبين من الإعدادات ← المستخدمون ← إضافة مستخدم (الدور = "مندوب مبيعات"). اسم المستخدم لازم يطابق الاسم اللي بيظهر في تعيين العملاء.

شغل المندوب — التوزيع التلقائي والمهام

إزاي العميل يوصل للمندوب الصح وميضيعش

🎯 التوزيع التلقائي للعملاء

مفيش توزيع يدوي. أول ما عميل يوصل تصنيف Warm (أو يقفز لـ Hot) والسيستم يلاقيه لسه مش معيّن لحد → بيوزّعه تلقائياً على المندوب الأقل عملاء نشطين.

العميل يبقى Warm → السيستم يدّيه للمندوب اللي عنده أقل عدد عملاء
المندوب يلاقيه فوراً في تاب "عملائي" (مش محتاج حد يعمل حاجة)
للتحكم: الإعدادات فيها auto_assign_enabled (شغّال افتراضياً — خلّيه false لو عايز توزيع يدوي)
🎯 ليه Warm مش Hot؟ عشان المندوب يمسك العميل وهو لسه بيهتم — مايستناش لما يبقى ساخن جداً ويكون اتأخر. والتوزيع بالعدالة (الأقل عملاء) عشان مفيش مندوب يتزنق والباقي فاضي.

🔔 المهام والتذكيرات

عشان مفيش عميل قال "كلّمني بعدين" ويضيع — فيه نظام مهام كامل:

تلقائي من المكالمة: في جلسة المكالمة يضغط المندوب "اتصل لاحقاً" → يظهر اختيار تاريخ + ملاحظة → تتعمل مهمة لوحدها
مانيوال من صفحة العميل: في ملف أي عميل → قسم "المهام والتذكيرات" → "أضف تذكير" (تاريخ + ملاحظة)
تاب "مهامي": المندوب يشوف كل مهامه — 🔴 متأخرة / 🟡 النهاردة / عادية — وزرار "تم" يقفلها
💡 الفايدة المندوب يفتح تاب "مهامي" الصبح يلاقي قايمة مرتّبة بكل اللي لازم يكلّمهم النهاردة — المتأخر فوق بالأحمر. مفيش عميل بيقع من الشبكة.
